# PLATFORM LISTING PLAYBOOK (VITO)

Обновлено: 2026-03-04  
Назначение: единый стандарт заполнения карточек товара (поля, SEO, теги, обложки, превью) для автопубликации.

## 1) Базовый стандарт карточки (обязательно)
- `title/name`: конкретная польза + формат + ниша.
- `short_description/summary`: 1-2 предложения, что получит покупатель.
- `description/long_description`: проблема -> решение -> состав -> результат -> CTA.
- `price`: тестовая цена и боевая цена разделяются флагами окружения.
- `category`: обязательная категория платформы.
- `tags`: только релевантные, без повторов и мусорных слов.
- `cover_path`: главная обложка.
- `thumb_path`: квадратный миниатюрный превью-кадр.
- `pdf_path` или основной файл товара.
- `preview_paths`: 2-6 скринов/превью (где платформа поддерживает).

## 2) SEO-структура (детерминированно, без лишних LLM-трат)
- Заголовок: 8-100 символов (или лимит платформы), ключ в первых словах.
- Короткое описание: до 160-220 символов, 1 ключ + ценность.
- Длинное описание:
  - 1 блок: кому продукт и какая боль решается.
  - 2 блок: что внутри (буллеты).
  - 3 блок: результат после использования.
  - 4 блок: ограничения/условия.
  - 5 блок: CTA.
- Теги: ключевые интенты покупки (не абстрактные слова).
- Дедупликация тегов и контроль длины/количества.

## 3) Лимиты по платформам (внедрены в optimizer)
- Gumroad:
  - title: 8-100
  - tags: до 5, до 32 символов
  - description min: 120
- Etsy:
  - title: 8-140
  - tags: до 13, до 20 символов
  - description min: 180
- Ko-fi:
  - title: 6-120
  - tags: до 10, до 32 символов
  - description min: 120
- Amazon KDP (через браузерный flow):
  - title: 8-200
  - tags/keywords: до 7, до 50 символов
  - description min: 200

## 4) Графика и превью
- Cover: читаемый крупный заголовок, 1 главный бенефит, контраст.
- Thumb: квадратный, минимум текста, фокус на узнаваемости.
- Preview: реальные страницы/скрины содержимого, не мокап-спам.
- Все превью должны подтверждать ценность и снижать сомнения перед покупкой.

## 5) Правила безопасности публикации
- Редактирование старых листингов только по явному запросу владельца.
- Для update обязателен `owner_edit_confirmed=true` + `target_*`.
- В режиме create запрещен скрытый fallback в старые листинги.

## 6) Автогейтинг качества перед публикацией
- `seo_score` (0-100) считается по факту заполнения:
  - корректный title
  - длинное описание
  - достаточные теги
  - категория
  - файл + визуальные ассеты
- `publish_ready=true` при `seo_score >= 80`.
